## What is tsctl?

tsctl is a **declarative infrastructure-as-code CLI** for managing Typesense collections, aliases, synonyms, overrides, and API keys. It brings the familiar Terraform plan/apply workflow to Typesense, making it easy to version control your search infrastructure and safely deploy changes across environments.

## Key Features

### Infrastructure as Code
Define your entire Typesense configuration in version-controlled files. Use TypeScript for full type safety, or choose JSON/YAML for simplicity.

### Safe Deployments
The plan/apply workflow shows you exactly what will change before making any modifications to your Typesense instance.

### Drift Detection
Detect when resources have been modified outside of tsctl, helping you maintain consistency across environments.

### Zero-Downtime Migrations
Blue/green deployment pattern for collection schema updates, enabling schema changes without downtime.

### Import Existing Resources
Start using tsctl with your existing Typesense instance - import all resources into managed state with a single command.

## Why tsctl?

**Version Control Your Search Infrastructure**  
Treat your Typesense configuration like code - commit it, review it, and track changes over time.

**Team Collaboration**  
Clear audit trail of who changed what and when. Review infrastructure changes in pull requests.

**Consistent Environments**  
Ensure development, staging, and production stay in sync. Detect and prevent configuration drift.

**Safe Deployments**  
Never guess what will happen - preview all changes before they're applied to your production instance.
