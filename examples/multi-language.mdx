## Overview

This example demonstrates how to build a multi-language search system with:
- Language-specific fields with locale and stemming
- Per-language synonym sets using Typesense 30.0+ global synonym sets
- Fallback to English for missing translations
- Search across multiple languages

## Key Concepts

### Locale-Specific Fields

The `locale` property enables language-specific text processing:

```typescript
{ 
  name: "title_en", 
  type: "string",
  locale: "en",   // English language processing
  stem: true,      // Enable stemming
}
```

Supported locales:
- `en` - English
- `es` - Spanish
- `fr` - French
- `de` - German
- `it` - Italian
- `pt` - Portuguese
- `ru` - Russian
- `zh` - Chinese
- `ja` - Japanese
- `ko` - Korean
- And many more...

### Stemming

Stemming reduces words to their root form for better search matching:

```typescript
{ 
  name: "description_en", 
  type: "string",
  locale: "en",
  stem: true,  // "running" → "run", "phones" → "phone"
}
```

Searching for "phones" will match documents containing "phone", "phones", "phoning", etc.

### Global Synonym Sets (Typesense 30.0+)

Global synonym sets are reusable across multiple collections:

```typescript
synonymSets: [
  {
    name: "global_synonyms",
    items: [
      {
        id: "en-phone",
        locale: "en",
        synonyms: ["phone", "mobile", "smartphone"],
      },
      {
        id: "es-phone",
        locale: "es",
        synonyms: ["teléfono", "móvil", "celular"],
      },
    ],
  },
]
```

Link to collections:

```typescript
{
  name: "products_multilang",
  fields: [...],
  synonym_sets: ["global_synonyms"],
}
```

### Legacy Per-Collection Synonyms (Typesense < 30.0)

For older versions, use per-collection synonyms:

```typescript
synonyms: [
  {
    id: "en-phone",
    collection: "products_multilang",
    locale: "en",
    synonyms: ["phone", "mobile", "smartphone"],
  },
  {
    id: "es-phone",
    collection: "products_multilang",
    locale: "es",
    synonyms: ["teléfono", "móvil", "celular"],
  },
]
```

### English Search

```bash

  -d 'q=phone&query_by=title_en,description_en' \
```

## Application Integration

Dynamic language selection in your application:

```javascript
const searchByLanguage = async (query, language) => {
  const languageFields = {
    en: 'title_en,description_en',
    es: 'title_es,description_es,title_en,description_en', // fallback to English
    fr: 'title_fr,description_fr,title_en,description_en',
    de: 'title_de,description_de,title_en,description_en',
  };

  const response = await fetch(
    `https://your-typesense.com/collections/products_live/documents/search`,
    {
      method: 'POST',
      headers: {
        'X-TYPESENSE-API-KEY': searchApiKey,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        q: query,
        query_by: languageFields[language] || languageFields.en,
        facet_by: 'category,brand',
        filter_by: 'in_stock:true',
      }),
    }
  );

  return response.json();
};

// Usage
const results = await searchByLanguage('teléfono', 'es');
```

## Best Practices

### 1. Always Include English as Fallback

Make English required and other languages optional:

```typescript
{ name: "title_en", type: "string" },  // Required
{ name: "title_es", type: "string", optional: true },  // Optional
```

### 2. Use Global Synonym Sets

If using Typesense 30.0+, prefer global synonym sets for easier management:

```typescript
synonym_sets: ["global_synonyms"]
```

### 3. Apply Consistent Stemming

Enable stemming for all searchable text fields:

```typescript
{ name: "title_en", type: "string", locale: "en", stem: true }
```

### 4. Structure Field Names Consistently

Use a consistent naming pattern:

```
{fieldname}_{locale}
```

Examples: `title_en`, `title_es`, `description_en`, `description_fr`
